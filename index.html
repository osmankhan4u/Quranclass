Current Challenges

Understanding the pain points in our current system

Operational Challenges

  • Manual enrollment: slow, inconsistent, and error-prone via WhatsApp.
  • Excel data storage: difficult to manage at scale, prone to mistakes.
  • Manual Zoom links: leads to confusion and missed classes.
  • Fragmented systems: no unified place for notes, revision, or quizzes.

Learning Challenges

  • Multiple apps: Zoom, WhatsApp, Google Drive, Forms, Seerah books.
  • No learning path: students can’t see where they are in the journey.
  • No reminders: missing personalized nudges or revision tracking.
  • Scattered notes: reflections often lost or stored randomly.

Teacher & Coordinator Challenges

  • No clarity: who revised, listened to audio, or completed tasks.
  • Manual tracking: high workload with limited oversight.
  • No insights: hard to identify students needing support.
  • No optimization: difficult to tailor teaching based on data.

"These challenges highlight the need for a unified digital platform that streamlines operations while enhancing the learning experience."

Enrollment Flow (Digitalized)

Digital Enrollment & Batch Assignment

Structured, scalable intake that keeps every stakeholder aligned

Step 1

Online Enrollment Form

  • Students complete a structured application with name, age, country, and motivation.
  • Availability & preferred time zones captured for smart scheduling.

Step 2

Review & Decision

  • Admins and organizers review submissions together.
  • Statuses applied with one click: Accepted, Waiting List, or Rejected.

Step 3

Batch Assignment

  • Assign batch codes (e.g., UAE-B01) with teacher and coordinator.
  • Set class day, time, and timezone for smooth cross-region operations.

Step 4

Student Onboarding

  • Students receive credentials instantly with login instructions.
  • Dashboard displays batch info, class timing, and expectations.

Stakeholder Reasoning

Why Digitize Intake?

Automating enrollment removes manual overhead, enforces consistent data capture, and makes adding new batches or regions as simple as duplicating a workflow.

Security & Compliance

Security, Ethics & Compliance

Policies rooted in amanah (trust) ensure every class upholds privacy, responsibility, and safety.

Camera Mandatory Policy

Etiquette Enforcement

  • Camera-on requirement is built into the class guidelines.
  • Communicated clearly throughout the UI and reminders before class.

Single Active Session

One Account · One Student

  • Session management prevents concurrent logins and account sharing.
  • Guarantees each enrolled learner participates personally.

Geo-Restriction

Regional Integrity

  • Optional policy to limit access by country or region.
  • Ensures batches remain region-specific where required.

Data Protection

Secure by Design

  • End-to-end HTTPS for all traffic.
  • Sensitive personal data encrypted at rest.

Audio Protection

Stream-Only Control

  • Class recordings available via protected streaming.
  • No downloads to prevent unauthorized sharing.

Audit Logging

Traceable Interactions

  • Joins, task submissions, and quiz attempts logged centrally.
  • Supports quality control and incident response.

Stakeholder Reasoning

Ethics Backed by Technology

Aligns with amanah, privacy, and responsibility while ensuring the program's integrity remains uncompromised.

Zoom Integration (Technical View)

Zoom Integration — Current & Future

Progressive rollout ensures we start reliable and build toward deeper insight.

Phase 1 — Now

Core Link Flow

  • Store Zoom link securely inside the platform.
  • “Join Class” button redirects directly to Zoom.
  • Attendance marked when a student clicks Join.

Phase 2 — Planned

API & Webhooks

  • Integrate Zoom APIs to track join/leave events.
  • Calculate actual attendance duration.
  • Sync data with dashboards for accuracy.

Phase 3 — Optional

Embedded Experience

  • Embed Zoom via Zoom Web SDK inside the web app.
  • Maintain security policies while reducing context switching.

Stakeholder Reasoning

Progressive Improvement

Start simple and reliable, then add deeper accuracy and insight as the program matures.
